# 管理后台规格

PC Web，灰蓝专业风格，面向平台内部运营人员。

---

## 模块清单

### 运营看板
- **设备状态**（实时，30s 轮询）：在线设备数、充电中端口数、活跃故障工单数
- **经营数据**（T-1，凌晨跑批）：昨日营收/订单数/用电量 + 环比增幅
- **图表**：近 7 日营收柱状图、订单量折线图
- **站点排行**：昨日营收 TOP 5，含订单数和端口利用率

### 站点管理
字段：编号、名称、地址、充电中（当前/总口数）、利用率（实时计算）、待处理工单数、计费规则  
操作：新增、编辑、查看设备、更换计费规则

### 设备管理
字段：设备编号（`CAB-A01`）、所属站点、端口数、设备状态、上次维护日期、计费规则、待处理工单数（角标）  
操作：添加、详情、报修（跳工单新建页，预填设备信息）、维护开关（切换 `maintenance_mode`）、设置计费规则  
筛选维度：**设备状态**（在线 / 离线 / 维护中 / 全部），单个下拉  
展示状态：**在线 / 离线 / 维护中**（由 `is_online` + `maintenance_mode` 派生，不落库；不再有「整机故障」「部分可用」）  
故障感知：通过工单角标体现，设备列表中有待处理工单的设备显示角标提示  
**API 筛选参数**：`is_online`（boolean?）、`maintenance_mode`（boolean?），详见 `docs/data-model.md` 设备状态节。

### 故障工单
字段：工单号（`WO-YYYY-XXXX`）、设备、站点、故障类型、优先级（高/中/低）、状态、负责人  
操作：新建、派工、详情

### 充电订单
筛选字段（每个独立 input）：订单号、用户昵称、手机号、站点（下拉）、状态（下拉）、今日/全部  
操作：查看详情、导出

### 充电会话
独立列表，与订单分离。可查无关联订单的异常会话（设备故障未触发结算的记录）。  
字段：会话 ID、关联订单 ID、设备/端口、用电量、峰值功率、结束原因

### 计费配置
- 计费规则版本化：草稿 → 发布 → 自动递增版本号（`v1.0` → `v1.1`）
- 规则字段：电费单价（¥/度）、4 档位服务费（¥/h）、超时占用费、最低预存余额
- 规则绑定粒度：**设备级**（优先）> 站点级
- 新版本生效时机：**用户下次开启充电时**，进行中会话不受影响
- 从规则侧可批量添加/移除关联设备；从设备侧也可单独更换规则，两侧同步

### 用户管理
字段：用户 ID（`U100XXX`）、昵称、手机号（脱敏）、注册日期、累计消费、账户余额、状态  
操作：详情、调整余额；按昵称/手机号/状态筛选、导出

### 优惠券管理
券类型：满减券 / 折扣券 / 赠金券  
字段：券名称、发行数、已领数、有效期、状态（进行中/即将到期/已结束）  
叠加：券 + 月卡可叠加；券 + 平台活动不可同享

### 报表中心
4 个子 Tab，支持按**时间段**（近 7/30/90 日）和**站点**筛选，均可导出。

| Tab | 核心 KPI | 图表 |
|---|---|---|
| 收入报表 | 服务费收入、净服务费、优惠抵扣、代收电费 | 服务费趋势折线、收入构成饼图、站点收入排行 |
| 订单报表 | 订单量、完成率、平均客单价、异常终止率 | 订单量趋势柱状、状态分布饼图、高峰时段分布、终止原因横条 |
| 设备报表 | 端口平均利用率、平均充电时长、本期故障工单数 | 各站端口利用率横条、利用率趋势折线 |
| 用户报表 | 注册用户（累计）、活跃用户、本期新增、复购率 | 新增用户趋势、消费分层饼图、充电高峰时段 |

**指标定义说明**（每个指标卡片均有 tooltip）：
- **代收电费**：过路款，代收代付，不计入平台收入
- **复购率**：本期充电 ≥2 次用户数 ÷ 本期活跃用户数
- **异常终止率**：非用户主动操作导致中途终止的订单占比
- **端口利用率**：充电时长 ÷ 可用总时长（低于 30% 可考虑缩减投入）
