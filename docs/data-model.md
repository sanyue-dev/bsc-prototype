# 数据模型 · 计费逻辑 · 状态机

---

## 核心实体关系

```
stations ──< devices ──< ports
stations >── billing_rules ──< devices  （设备级覆盖站点级）
charging_orders 1──n charging_sessions  （正常 1:1，断线重连可 1:n）
```

---

## 实体字段

### stations（站点）
| 字段 | 说明 |
|---|---|
| `id` | 如 `S001` |
| `name` / `address` | 展示给用户的名称和地址 |
| `operator_id` | 归属运营商 |
| `billing_rule_id` | 默认计费规则（设备级可覆盖）|

> 站点不存储枚举状态字段。管理后台展示的**利用率**（充电中端口数 ÷ 总端口数）和**待处理工单数**均为实时计算/聚合，不落库。

### devices（设备/主机箱）
| 字段 | 说明 |
|---|---|
| `id` | 如 `CAB-A01` |
| `station_id` | 所属站点 |
| `port_count` | 端口总数，通常 4～12 |
| `billing_rule_id` | 设备级规则，`null` 则继承站点 |
| `is_online` | boolean，心跳超时自动翻转，系统写入 |
| `maintenance_mode` | boolean，运营人员手动开关，人工写入 |
| `last_heartbeat_at` | 最后心跳时间戳 |

### ports（端口/插座）
| 字段 | 说明 |
|---|---|
| `id` | 如 `A-03` |
| `device_id` | 所属设备 |
| `status` | 空闲 / 使用中 / 故障（**无在线状态**）|

### billing_rules（计费规则）
| 字段 | 说明 |
|---|---|
| `id` | 如 `R001` |
| `current_ver` | 当前版本，如 `v1.2` |
| `status` | 草稿 / 已发布 |
| `versions[]` | 版本历史：`elec_price` / `tier_fees[4]` / `over_time_fee` / `min_balance` |

### charging_sessions（充电会话，高频写入）
| 字段 | 说明 |
|---|---|
| `id` | `SES-YYYYMMDD-XXX` |
| `order_id` | 关联订单，异常时可为空 |
| `device_id` / `port_no` | 设备 + 端口 |
| `kwh` / `peak_power` | 用电量（度）/ 峰值功率（W）|
| `stop_reason` | 正常完成 / 用户停止 / 超时断电 / 设备异常 / 充电中 |

### charging_orders（充电订单，低频写入）
| 字段 | 说明 |
|---|---|
| `id` | `ORD-YYYYMMDD-XXXX` |
| `session_id` | 关联充电会话 |
| `elec_fee` / `service_fee` / `discount` / `actual_amount` | 费用拆分 |
| `pricing_rule_snapshot` | 规则名+版本快照，结算后不可变 |
| `status` | 待结算 / 已完成 / 已退款 / 异常（**无「充电中」**）|

## 计费逻辑

**总费用 = 服务费（功率档位 × 时长）+ 电费（度数 × 单价）**，结束时一次结算。

### 功率档位
| 档位 | 范围 | 示例 |
|---|---|---|
| 档位一 | 0–100 W | ¥0.40/h |
| 档位二 | 101–200 W | ¥0.50/h |
| 档位三 | 201–300 W | ¥0.60/h |
| 档位四 | 301–400 W | ¥0.70/h |

### 计费周期
- 可配置（预设 3/5/6/10 分钟，支持自定义 1–60）
- 不足一个周期**向上取整**
- 实时检测功率，按当前功率对应档位计费

### 自动断电
- 电流低于阈值（充满判定）→ 自动断电
- 超流保护（> 5A）→ 自动断电 + 推送告警
- 断电后占用不额外收费（可配置超时占用费，断电 N 分钟后开始计）

### 优惠叠加
- 优惠券 + 月卡 ✅ 可叠加
- 优惠券 + 平台活动 ❌ 不可同享
- 每次仅可用一张券

## 状态机

### 端口状态
```
空闲 ──[用户启动]──> 使用中 ──[结束/停止]──> 空闲
任意 ──[硬件异常]──> 故障 ──[维修恢复]──> 空闲
```

> ⚠️ **硬件约束：设备不支持主动上报端口故障。**  
> `ports.status = 故障` 只能由**人工写入**（运营人员在后台标记）或**超流保护触发**（电流 > 5A 自动断电时系统写入）。  
> 设备无法感知短路、过温、接触不良等端口级故障并主动推送事件，因此：  
> - 故障工单**不可自动创建**，只能人工发现后手动报修  
> - 端口故障发现依赖巡检或用户反馈，存在滞后  
> - 后续如升级固件支持 MQTT 故障事件上报，可解锁自动工单能力

### 设备状态

设备状态由两个字段决定：`is_online`（系统写，心跳超时自动翻转）和 `maintenance_mode`（人工写，运营人员手动开关）。

| 展示状态 | 条件 | 说明 |
|---|---|---|
| 离线 | `is_online = false` | 心跳超时，不可用 |
| 维护中 | `is_online = true` && `maintenance_mode = true` | 计划停机，不触发离线告警，不自动创建工单 |
| 在线 | `is_online = true` && `maintenance_mode = false` | 正常运行 |

> **不再有「整机故障」「部分可用」状态。** 由于设备不支持主动上报端口故障（见端口状态约束），基于端口故障数派生的设备状态无法准确反映真实健康情况。故障感知统一通过**故障工单**体现：设备列表中以工单角标提示「有待处理工单」。

#### 设备列表筛选 API 参数

| UI 选择 | `is_online` | `maintenance_mode` |
|---|---|---|
| 在线 | `true` | `false` |
| 维护中 | `true` | `true` |
| 离线 | `false` | — |
| 全部 | — | — |

**后端接口参数**（均为可选）：

| 参数 | 类型 | 后端处理 |
|---|---|---|
| `is_online` | boolean? | 直接 `WHERE is_online = ?` |
| `maintenance_mode` | boolean? | 直接 `WHERE maintenance_mode = ?` |

### 订单状态
```
待结算（充电进行中）──[充电结束]──> 已完成 ──[申请退款]──> 已退款
                  ──[异常终止]──> 异常
```
> ⚠️ 订单状态中**不存在「充电中」**，充电进行中的订单状态为「待结算」。

### 工单状态
```
待处理 ──[派工]──> 处理中 ──[修复]──> 已完成
                          ──[关闭]──> 已关闭
```
