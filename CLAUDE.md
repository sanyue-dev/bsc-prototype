# 蓝鲨充电 · 项目规范

## 项目背景

**蓝鲨充电是两轮电动车充电柜运营平台，不是汽车充电桩。**

- 充电对象：电动自行车、电动摩托车（两轮）
- 充电设备：一台主机箱引出多个充电插座，每个插座对应一个充电端口；用户将自己的充电器插入插座完成充电（类似共享插线板，无柜门）
- 典型场景：小区楼下、菜市场、商场停车场等公共场所
- 计费方式：按时长或按电量计费，单次金额较小（几毛到几块钱）
- 用户端：微信小程序扫码启用插座
- 运营端：Web 管理后台（本项目）

设计时请勿出现汽车、充电枪、充电桩、充电柜门等与汽车或储物柜相关的概念和图示。

## 设备与端口术语规范

- **设备（device）**：一台主机箱，拥有若干充电端口（通常 4~10 个插座），主机整体与服务器保持心跳连接
- **端口（port）**：主机箱上的单个充电插座，状态分为：空闲、使用中、故障
- **「端口在线率」是错误概念**：端口没有独立的网络连接，不存在「在线/离线」；连接状态属于设备（主机）层级，应使用「设备在线率」
- 正确用法示例：
  - ✅ 设备在线率 = 心跳正常的设备数 ÷ 全部设备数
  - ✅ 端口利用率 = 使用中端口数 ÷ 全部端口数（或充电时长/可用时长）
  - ✅ 端口故障率 = 故障端口数 ÷ 全部端口数
  - ❌ 端口在线率（不存在此概念）

## 原型文件结构

本项目包含两个独立的原型入口：

### index.html — 小程序端原型
模拟微信小程序 UI，使用 iOS 手机框架展示。根据 Tweaks 面板中的「端」切换，有两种身份视图：

- **车主端** (`role: "owner"`)：普通用户，主要流程为扫码 → 找站点 → 确认充电 → 充电中 → 完成，共 18 个屏幕
- **运营商端** (`role: "operator"`)：加盟运营商，查看自己名下站点的数据看板、设备监控、故障工单等，共 5 个屏幕

相关源文件：`app.jsx` `ui.jsx` `screens-core.jsx` `screens-charging.jsx` `screens-h5.jsx` `screens-operator.jsx`

#### UniApp 开发规范（设计原型需遵守）
index 端未来由程序员用 **UniApp** 实现，目标平台为 **H5 + 微信小程序**，设计原型应遵循 UniApp 最佳实践：

- **顶部标题栏使用原生导航栏**：不要自绘 header div，标题通过页面配置 `navigationBarTitleText` 设置，原型中用系统原生风格的标题栏模拟（即 iOS 状态栏下方紧接着的原生 NavBar，含返回箭头和居中标题）
- **页面内容区顶部留出安全距离**：内容从导航栏底部开始，不与状态栏/导航栏重叠
- **底部 Tab Bar 使用原生 TabBar 风格**：不要自绘底部导航，使用系统风格 tab bar
- **底部安全区适配**：页面底部内容需留出 `env(safe-area-inset-bottom)` 安全距离
- **避免使用仅 Web 支持的 CSS**：如 `position: fixed` 在小程序中有限制，尽量用 flex 布局替代

### admin.html — Web 管理后台
面向平台管理员的 PC 端后台，包含：运营看板、充电订单、充电会话、站点管理、设备管理、故障工单、计费配置、用户管理、优惠券管理。

相关源文件：`admin-components.jsx` `admin-login.jsx` `admin-screens-1.jsx` `admin-screens-2.jsx` `admin-screens-3.jsx`

### landing.html — 落地页 / 宣传页
面向 C 端用户和潜在加盟商的对外宣传页，风格与小程序端保持一致（eco 绿主色、亮色系）。

## 视觉风格分层规范

本项目三个端的视觉风格**刻意分层**，不要求全局统一：

| 端 | 受众 | 风格定位 |
|---|---|---|
| `landing.html` | C 端用户、潜在加盟商 | 品牌亲和，与小程序一致：eco 绿（`#1A9E6E`）主色、亮色系 |
| `index.html` | 车主 / 运营商 | 同上，`THEMES.eco` 配色，轻量友好 |
| `admin.html` | 平台内部运营人员 | 专业中性，灰蓝系，适合高密度数据表格；**无需与小程序配色一致** |

> 管理后台保持自身专业配色即可，Logo / 品牌名称出现在侧边栏即满足品牌认知需求。

## 搜索 Input 规范

**每个 `<input>` 只负责一个字段的搜索/筛选。**

- 禁止用单个 input 同时模糊搜索多个字段（如同时搜索"订单号"和"用户名"）
- 每个搜索框必须有明确的 `placeholder` 标注它对应的字段，例如：
  - `placeholder="搜索订单号"` → 只查 order_id
  - `placeholder="搜索用户手机号"` → 只查 phone
  - `placeholder="搜索设备编号"` → 只查 device_id
- 多字段筛选用多个独立 input 并排排列，不合并
- 每个 input 的 `name` / `data-field` 属性应与后端字段名一一对应

### 示例（正确）
```jsx
<Input placeholder="搜索订单号"   data-field="order_id" />
<Input placeholder="搜索用户手机" data-field="phone" />
```

### 示例（错误）
```jsx
<Input placeholder="搜索订单号 / 用户手机 / 设备编号" /> {/* ❌ 一个 input 多个字段 */}
```
